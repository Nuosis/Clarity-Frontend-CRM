## Executive Summary

This change request defines the REST API endpoints required for task CRUD operations, timer management, and financial record generation. The API will replace the current FileMaker-based implementation and serve as the backend for Supabase-only task management.

**Impact:**
- **Database:** Requires `tasks` and `task_timers` tables in Supabase (schema defined below)
- **API:** 12 new REST endpoints with HMAC authentication
- **Performance:** Improved query performance with proper indexing
- **Data Volume:** ~10,000 existing tasks to migrate (estimate)

**Dependencies:**
- `projects` table must exist (verified)
- `customer_sales` table must exist (verified)
- `customers` table must exist (verified)
- Organization structure must be stable
- Staff/team tables must exist

## Database Schema

### 1. Tasks Table

```sql
CREATE TABLE IF NOT EXISTS tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  -- Core fields
  task_id UUID NOT NULL UNIQUE, -- Frontend-generated UUID (maps to FileMaker __ID)
  task TEXT NOT NULL, -- Task name/description (max 200 chars)
  type TEXT, -- Task type (e.g., "General")

  -- Relationships
  project_id UUID NOT NULL, -- FK to projects.id
  staff_id UUID NOT NULL, -- FK to staff/team members
  organization_id UUID NOT NULL, -- FK to organizations.id

  -- Status and priority
  is_completed BOOLEAN NOT NULL DEFAULT FALSE,
  priority TEXT DEFAULT 'active', -- 'active', 'high', 'low'

  -- Timestamps
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  completed_at TIMESTAMPTZ, -- When task was completed

  -- Constraints
  CONSTRAINT tasks_task_length CHECK (char_length(task) > 0 AND char_length(task) <= 200),
  CONSTRAINT tasks_priority_check CHECK (priority IN ('active', 'high', 'low'))
);

-- Indexes for performance
CREATE INDEX IF NOT EXISTS idx_tasks_project_id ON tasks(project_id);
CREATE INDEX IF NOT EXISTS idx_tasks_staff_id ON tasks(staff_id);
CREATE INDEX IF NOT EXISTS idx_tasks_organization_id ON tasks(organization_id);
CREATE INDEX IF NOT EXISTS idx_tasks_task_id ON tasks(task_id);
CREATE INDEX IF NOT EXISTS idx_tasks_is_completed ON tasks(is_completed);
CREATE INDEX IF NOT EXISTS idx_tasks_created_at ON tasks(created_at DESC);

-- Composite index for common queries
CREATE INDEX IF NOT EXISTS idx_tasks_project_status
  ON tasks(project_id, is_completed, created_at DESC);

-- Update trigger for updated_at
CREATE OR REPLACE FUNCTION update_tasks_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  IF NEW.is_completed = TRUE AND OLD.is_completed = FALSE THEN
    NEW.completed_at = NOW();
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER tasks_updated_at_trigger
  BEFORE UPDATE ON tasks
  FOR EACH ROW
  EXECUTE FUNCTION update_tasks_updated_at();
```

**Field Mappings from FileMaker:**
- `task_id` ← FileMaker `__ID` (UUID generated by frontend)
- `task` ← FileMaker `task`
- `type` ← FileMaker `type`
- `project_id` ← FileMaker `_projectID` (need to map to Supabase project UUID)
- `staff_id` ← FileMaker `_staffID` (need to map to Supabase staff UUID)
- `is_completed` ← FileMaker `f_completed` (0/1 → boolean)
- `priority` ← FileMaker `f_priority`
- `created_at` ← FileMaker `~creationTimestamp`
- `updated_at` ← FileMaker `~modificationTimestamp`

### 2. Task Timers Table

```sql
CREATE TABLE IF NOT EXISTS task_timers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  -- Relationships
  task_id UUID NOT NULL, -- FK to tasks.id
  project_id UUID NOT NULL, -- FK to projects.id
  customer_id UUID NOT NULL, -- FK to customers.id
  staff_id UUID NOT NULL, -- FK to staff
  organization_id UUID NOT NULL, -- FK to organizations.id

  -- Timer data
  start_time TIMESTAMPTZ NOT NULL,
  end_time TIMESTAMPTZ,

  -- Work description
  work_performed TEXT,

  -- Time calculations
  billable_hours NUMERIC(10,2), -- Calculated when timer stops
  adjustment_seconds INTEGER DEFAULT 0, -- Manual time adjustments

  -- Financial integration
  financial_record_id UUID, -- FK to customer_sales.id (if applicable)
  is_billable BOOLEAN DEFAULT TRUE,

  -- Timestamps
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

  -- Constraints
  CONSTRAINT task_timers_time_check CHECK (end_time IS NULL OR end_time > start_time),
  CONSTRAINT task_timers_adjustment_increment CHECK (adjustment_seconds % 360 = 0) -- 6-minute increments
);

-- Indexes
CREATE INDEX IF NOT EXISTS idx_task_timers_task_id ON task_timers(task_id);
CREATE INDEX IF NOT EXISTS idx_task_timers_project_id ON task_timers(project_id);
CREATE INDEX IF NOT EXISTS idx_task_timers_customer_id ON task_timers(customer_id);
CREATE INDEX IF NOT EXISTS idx_task_timers_staff_id ON task_timers(staff_id);
CREATE INDEX IF NOT EXISTS idx_task_timers_organization_id ON task_timers(organization_id);
CREATE INDEX IF NOT EXISTS idx_task_timers_start_time ON task_timers(start_time DESC);
CREATE INDEX IF NOT EXISTS idx_task_timers_financial_record_id ON task_timers(financial_record_id);

-- Active timer queries
CREATE INDEX IF NOT EXISTS idx_task_timers_active
  ON task_timers(staff_id, start_time DESC)
  WHERE end_time IS NULL;

-- Update trigger
CREATE OR REPLACE FUNCTION update_task_timers_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();

  -- Calculate billable hours when timer stops
  IF NEW.end_time IS NOT NULL AND OLD.end_time IS NULL THEN
    NEW.billable_hours = ROUND(
      (EXTRACT(EPOCH FROM (NEW.end_time - NEW.start_time)) + NEW.adjustment_seconds) / 3600.0,
      2
    );
  END IF;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER task_timers_updated_at_trigger
  BEFORE UPDATE ON task_timers
  FOR EACH ROW
  EXECUTE FUNCTION update_task_timers_updated_at();
```

**Field Mappings from FileMaker:**
- `start_time` ← Combine FileMaker `DateStart` + `TimeStart`
- `end_time` ← Combine FileMaker `DateStart` + `TimeEnd` (if stopped)
- `work_performed` ← FileMaker `Work Performed`
- `adjustment_seconds` ← FileMaker `TimeAdjust`
- `billable_hours` ← FileMaker `Billable_Time_Rounded`

### 7. Stop Timer

**Endpoint:** `POST /api/tasks/{task_id}/timer/stop`

**Description:** Stops an active timer and optionally creates a financial record.

**Path Parameters:**
- `task_id` (UUID) - Task ID

**Request Body:**
```json
{
  "timer_id": "timer-uuid",
  "work_performed": "Implemented authentication logic",
  "adjustment_seconds": 360,
  "save_immediately": false
}
```

**Required Fields:**
- `timer_id` (UUID) - Timer ID to stop

**Optional Fields:**
- `work_performed` (string) - Work description (default: "Time logged")
- `adjustment_seconds` (integer, multiple of 360) - Manual time adjustment
- `save_immediately` (boolean) - If true, use default description

**Response (200 OK):**
```json
{
  "timer": {
    "id": "timer-uuid",
    "task_id": "task-uuid",
    "project_id": "project-uuid",
    "customer_id": "customer-uuid",
    "staff_id": "staff-uuid",
    "organization_id": "org-uuid",
    "start_time": "2026-01-10T14:30:00Z",
    "end_time": "2026-01-10T16:45:00Z",
    "work_performed": "Implemented authentication logic",
    "billable_hours": 2.25,
    "adjustment_seconds": 360,
    "is_billable": true,
    "financial_record_id": "sales-record-uuid",
    "created_at": "2026-01-10T14:30:00Z",
    "updated_at": "2026-01-10T16:45:00Z"
  },
  "financial_record": {
    "id": "sales-record-uuid",
    "created": true
  }
}
```

**Business Logic:**
1. Validate `timer_id` exists and `end_time IS NULL`
2. Validate `adjustment_seconds` is multiple of 360 (6 minutes)
3. Set `end_time` to current timestamp
4. Set `work_performed` (use "Time logged" if `save_immediately = true`)
5. Calculate `billable_hours` = (end_time - start_time + adjustment_seconds) / 3600
6. Fetch project to check `f_fixedPrice` field
7. **If project is NOT fixed-price**: Create financial record in `customer_sales` table
   - `financial_id` = generate new UUID
   - `customer_id` = from project
   - `product_name` = format: "{CUSTOMERCAPS}:{ProjectFirstWord}"
   - `quantity` = `billable_hours`
   - `unit_price` = staff hourly rate (fetch from staff record)
   - `total_price` = `quantity * unit_price`
   - `date` = timer `start_time` date
   - `organization_id` = from timer
8. **If project IS fixed-price**: Skip financial record creation
9. Link `financial_record_id` to timer if created

**Error Responses:**
- `400 Bad Request` - Validation error (adjustment not multiple of 360)
- `404 Not Found` - Timer not found
- `409 Conflict` - Timer already stopped
- `500 Internal Server Error` - Database error

**Fixed-Price Detection:**
From `src/services/taskService.js:104-111`:
```javascript
const fixedPrice = parseFloat(financialRecord.response.data[0].fieldData["customers_Projects::f_fixedPrice"] || 0);
```

## Authentication

All API endpoints require HMAC-SHA256 authentication following the existing backend pattern.

**Implementation Reference:** `src/services/dataService.js:61-105`

**Request Header:**
```
Authorization: Bearer {signature}.{timestamp}
```

**Signature Generation:**
```javascript
const timestamp = Math.floor(Date.now() / 1000);
const message = `${timestamp}.${JSON.stringify(requestBody)}`;
const signature = HMAC-SHA256(message, SECRET_KEY);
```

**Backend Validation:**
1. Extract signature and timestamp from Authorization header
2. Verify timestamp is within 5 minutes of current time
3. Reconstruct message using timestamp and request body
4. Calculate expected signature using SECRET_KEY
5. Compare signatures (constant-time comparison)
6. Reject if signatures don't match or timestamp is stale

**Environment Variable:**
- `VITE_SECRET_KEY` - Shared secret for HMAC signing

## Validation Rules

### Task Validation

**Required Fields:**
- `task_id` - Must be valid UUID
- `task` - Must be 1-200 characters
- `project_id` - Must exist in `projects` table
- `staff_id` - Must exist in staff/team table
- `organization_id` - Must exist in `organizations` table

**Optional Fields:**
- `type` - Any string
- `priority` - Must be 'active', 'high', or 'low'

**Cross-Field Validation:**
- `project_id` must belong to `organization_id`
- `staff_id` must belong to `organization_id`

**Implementation Reference:** `src/services/taskService.js:265-376`

### Timer Validation

**Start Timer:**
- Staff cannot have more than one active timer
- Task must exist and not be deleted
- Task must belong to valid project

**Stop Timer:**
- Timer must exist and have `end_time = NULL`
- `adjustment_seconds` must be multiple of 360 (6 minutes)
- `work_performed` required unless `save_immediately = true`

**Adjustment Validation:**
From `src/services/taskService.js:619-622`:
```javascript
export function isValidTimerAdjustment(minutes) {
    // Only allow adjustments in 6-minute increments
    return minutes % 6 === 0;
}
```

Converted to seconds: `adjustment_seconds % 360 === 0`

## Business Logic

### 1. Timer Financial Record Generation

**Trigger:** Timer stop event

**Logic:**
1. Calculate billable hours from timer
2. Fetch project to check `f_fixedPrice` field
3. **IF project.f_fixedPrice > 0:** Skip financial record creation
4. **ELSE:** Create financial record in `customer_sales` table

**Financial Record Format:**
```javascript
{
  financial_id: uuidv4(), // New UUID
  organization_id: timer.organization_id,
  customer_id: project.customer_id,
  product_name: formatProductName(customer, project),
  quantity: timer.billable_hours,
  unit_price: staff.hourly_rate,
  total_price: timer.billable_hours * staff.hourly_rate,
  date: timer.start_time (date only),
  inv_id: null // Not invoiced yet
}
```

**Product Name Format:**
From `requirements/financial-records/BACKEND_CHANGE_REQUEST_001_FINANCIAL_RECORDS_MIGRATION.md:567`:
```
CUSTOMERCAPS:ProjectFirstWord
```
Example: `CLARITYBUSINESS:WebsiteRedesign`

**Code Reference:** `src/services/taskService.js:82-128`

---

### 2. Project Fixed-Price Detection

**Logic:**
```sql
SELECT f_fixedPrice FROM projects WHERE id = :project_id
```

If `f_fixedPrice > 0`, the project is fixed-price and timers should NOT generate financial records.

**Rationale:** Fixed-price projects have a predetermined cost, so individual timer entries should not create billable line items.

**Code Reference:** `src/services/taskService.js:104-111`

---

### 3. Task Sorting

**Sort Order:**
1. Active tasks (`is_completed = false`) before completed tasks
2. Within each group: Newest first (by `created_at DESC`)

**SQL Implementation:**
```sql
SELECT * FROM tasks
WHERE project_id = :project_id
ORDER BY
  is_completed ASC,
  created_at DESC
```

**Code Reference:** `src/services/taskService.js:645-658`
