## Changes Made

4. Provides consistent error handling with environment context

## Environment Detection

Environment is set during app initialization in `src/index.jsx`:

1. **SignIn Component** detects environment:
   - Checks for `window.FileMaker` or `window.FMGofer`
   - Calls `onFileMakerDetected()` or `onSupabaseAuth()`

2. **App Component** sets environment context:
   - FileMaker: `setEnvironmentContext({ type: ENVIRONMENT_TYPES.FILEMAKER, ... })`
   - Web App: `setEnvironmentContext({ type: ENVIRONMENT_TYPES.WEBAPP, ... })`

3. **Global State** maintains environment throughout session:
   - Stored in `AppStateContext` (React Context)
   - Accessible via `getEnvironmentContext()` from any API module

Web app environment uses these backend endpoints:

| Operation | Method | Endpoint | Description |
|-----------|--------|----------|-------------|
| List all | GET | `/contacts_api` | Fetch all customers |
| Get one | GET | `/contacts_api/{id}` | Fetch single customer |

| Update | PATCH | `/contacts_api/{id}` | Update customer |
| Delete | DELETE | `/contacts_api/{id}` | Delete customer |
| Filter | GET | `/contacts_api?f_active=1` | Query parameters for filtering |

## Authentication

### FileMaker Environment
- Uses FileMaker bridge authentication
- No additional headers required
- Authentication handled by fm-gofer

### Web App Environment
- Uses HMAC-SHA256 authentication
- Format: `Bearer {signature}.{timestamp}`
- Generated by `generateBackendAuthHeader()` in dataService
- Automatically added by axios interceptor
